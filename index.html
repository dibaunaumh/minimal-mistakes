The way we receive updates from Web sites & Mobile apps today, is based on an Assorted Feed - items arrive in a semantically random order, & we consume them sequentially, until we find something that interests us. This interface is just like the baggage carousel you use in airports, which can only work if you have a few dozen items, but cannot be effective for thousands of items. There exist interfaces in RL that do allow you to find & discover a few items of interest out of thousands of items, for example, supermarkets or stores, where items are organized in a space, which you learn over time & become very efficient in navigating & consuming. The Web Wide Matrix uses the Organized Space interface as its underlying abstraction, allowing users to consume much more information in much less time." core_aspects__access: - title: "The Access Model" excerpt: "Data & information is scattered in many islands today, making it difficult to discover, access, process & aggregate it. Many data providers today offer an API interface to their data, based on the REST guidelines, yet this access model wasn't designed for the amounts of information we have today, & doesn't provide us with any way to discover, process, combine & aggregate large amounts of information. The Web Wide Matrix offers a completely new abstraction for storing, organizing & accessing data, based on the concept of Composite Buildings. This new abstraction provides a simple standard way to access any piece of data in the world, enabling easy discovery, processing & aggregation of data." core_aspects__processing: - title: "The Processing Model" excerpt: "Every time an organization needs to process some big set or stream of data, it needs to invest a big effort in developing a full proprietary solution for processing this type of data for the organization needs. This takes a lot of time & resources, so only big companies can afford to build such solutions, & only for some specific data. This is similar to the situation we had before the introduction of the Web - it was very difficult to enable access between the computers of 2 organizations, because there wasn't a common abstraction & standard easy way to do that. The Web Wide Matrix introduces a new abstraction for processing large amounts of information, that tries to do what the Web did: make it dead simple & easy for anyone to process any type of data. This abstraction is based on generic workers that are given Training Courses for processing data, which will be available in common marketplaces.
